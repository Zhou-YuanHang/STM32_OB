# STM32 总线总结

> 根据与 Reasonix 的对话整理，2025

---

## 一、经典 CPU 总线模型（8086 / 51）回顾

- **地址总线** — 输出地址，选择访问的存储单元或外设
- **数据总线** — 传输数据（双向）
- **控制总线** — RD# / WR# 等时序控制信号
- 地址和数据是**独立的物理引脚**，需手动控制时序（如 `MOVX @DPTR, A`）

---

## 二、STM32 内部总线架构（现代演化版）

经典模型的"地址/数据/控制"三总线在 STM32 中演化为**总线矩阵 + 分层总线结构**：

### 2.1 Cortex-M 内核专用总线

| 总线 | 全称 | 作用 |
|------|------|------|
| **I-Bus** | Instruction Bus | 内核取指令专用路径 |
| **D-Bus** | Data Bus | 内核加载/存储数据专用路径 |
| **S-Bus** | System Bus | 内核访问外设和系统 SRAM |

### 2.2 AHB — 高级高性能总线

- 通常是系统主总线，频率高（与 HCLK 同步）
- 位宽 32/64/128 位
- 挂载：**Flash、SRAM、DMA、USB HS、Ethernet、CRC** 等
- 部分系列（F7/H7）还有 **AHB4 / AHB5** 扩展

### 2.3 APB — 高级外设总线（低速分支）

AHB 下分两个速度等级：

| 总线 | 速度 | 挂载的典型外设 |
|------|------|----------------|
| **APB1** | 慢（≤ 36–72 MHz） | USART2-5、SPI2/3、I2C1-3、TIM2-7、PWR、RTC、DAC |
| **APB2** | 快（同 HCLK） | USART1、SPI1、TIM1/8、ADC、GPIO、EXTI |

> 配置外设时钟时要查它是挂在 APB1 还是 APB2，两者最大频率不同。

### 2.4 AXI（H7 系列专有）

- Advanced eXtensible Interface
- 多通道 64 位并发
- 允许 CPU、DMA、GPU 等主设备同时访问不同 slave，不冲突

### 2.5 层级关系示意（以 F4 为例）

```
CPU Core
  ├─ I-Bus → I-Cache → Flash
  ├─ D-Bus → D-Cache → Flash / SRAM1
  └─ S-Bus → AHB ──┬─ SRAM / DMA / USB HS
                    ├─ APB1（慢速外设）
                    └─ APB2（高速外设）
```

## 三、"地址总线"概念在 STM32 中的映射

- 独立的**地址引脚**不复存在（地址和数据复用，由硬件自动管理）
- 但**地址的概念仍然存在**——每个外设寄存器在 **Memory Map** 中分配了唯一的基地址
- 你在代码里 `USART1->DR = data` 时，硬件自动将寄存器地址 + 读/写控制信号送上 AHB/APB
- 查芯片手册的 **Register map / Memory map** 表格就能看到谁在哪个地址

### 3.5 为什么说 STM32“没有独立的地址总线引脚”？

拿 8086 接外部 RAM 的场景对比：

| | 8086 / 51 | 大多数 STM32 |
|---|---|---|
| 地址引脚 | 芯片上有专门的引脚（如 A0–A15）直接连到 RAM 的地址引脚 | **没有**外部地址引脚——地址只在芯片**内部** AHB/APB 总线上跑 |
| 数据引脚 | 数据引脚与地址引脚分开，分别接到外设 | 数据线同样不对外暴露，通过协议接口（I²C/SPI/UART）传输 |
| 控制时序 | 手动拉 RD# / WR#，或 `MOVX @DPTR, A` | 读写寄存器时硬件自动产生时序，你只需 C 代码赋值 |
| 接外设 | 地址线 + 数据线分别连到外设芯片 | 通过串行协议（I²C/SPI/UART）或 FSMC/FMC |

**地址线退回到了芯片内部，不再作为引脚暴露出来。** 每个 STM32 引脚都是多功能复用的（如 PA0 可以是 GPIO / TIM1_CH1 / USART2_CTS / ADC12_IN0……），但没有一个引脚叫 `A0`、`A1`——地址信号在 AHB/APB 总线矩阵内部由硬件自动路由，你看不到它，也不需要手动控制。

#### 例外：FSMC / FMC 接口

在 STM32 的大容量型号（F407、F429、H743 等）上，**FSMC（Flexible Static Memory Controller）或 FMC** 会把地址线重新引出来：

- `A0–A25` — 地址线
- `D0–D15` — 数据线
- `NE#` — 片选
- `NWR#` — 写使能
- `NOE#` — 读使能

这些引脚通过 GPIO 复用模式配置出来，用途就是像 8086 一样直接接外部 SRAM / NOR Flash / LCD。**但这是特例，不是所有 STM32 都有这个功能。**

## 四、外设通信总线（对外接口）

STM32 的物理引脚跑的是**协议总线**，不是传统的地址线：

| 总线 | 物理线数 | 速度 | 特点 | 常见用途 |
|------|----------|------|------|----------|
| **USART/UART** | 2 (TX/RX) | ≤ 10 Mbps | 异步，全双工 | 串口调试、GPS、蓝牙 |
| **I²C** | 2 (SCL/SDA) | ≤ 1 MHz | 同步，多主，需上拉 | 传感器、OLED、EEPROM |
| **SPI** | 3-4 (SCK/MOSI/MISO/CS) | ≤ 40 MHz | 同步，全双工 | Flash、显示屏、SD 卡 |
| **CAN / CAN-FD** | 2 (CAN_H/L) | ≤ 1 Mbps / FD ≤ 8 Mbps | 差分，多主，带仲裁 | 汽车、工业控制 |
| **I²S** | 3-4 (CK/WS/SD/MCK) | 音频频率 | 音频用 SPI 变体 | 麦克风、音频 DAC |
| **USB FS/HS** | 2 (D+/D-) | 12 / 480 Mbps | Host 或 Device | 键盘、虚拟串口 |
| **SDIO/SDMMC** | 6-9 | ≤ 48 MHz | 4 位并行 | SD 卡、eMMC |
| **Ethernet MAC** | MII/RMII | 10/100 Mbps | 需外接 PHY | 网络通信 |
| **LIN** | 1 | ≤ 20 kbps | 低成本 CAN 替代 | 车门/车窗 |
| **SMBus / PMBus** | 2 | I²C 兼容 | 系统管理 | 电池管理 |

---

## 五、一句话总结

> **经典的地址/数据/控制总线 → STM32 内部缩成了 I-Bus / D-Bus / S-Bus + AHB → APB 分层总线矩阵。**
> 外部引脚全都变成了协议总线（I²C / SPI / UART / CAN 等），对大多数型号而言不再有独立的地址总线引脚。
> 地址的概念体现在 **Memory Map 的寄存器基地址**中，由硬件自动处理寻址和时序。
> **例外：** 带 FSMC/FMC 的型号（如 F407、F429、H743）可以通过 GPIO 复用把地址/数据线重新引出来，用于接外部 SRAM / NOR Flash / LCD。
