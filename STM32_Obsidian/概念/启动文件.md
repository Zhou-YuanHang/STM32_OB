## 一、用最通俗的话说——启动文件是干啥的？
### 打个比方：煮饭
假如你想煮一锅饭：

|        步骤        |     谁干的      |      对应到单片机      |
| :--------------: | :----------: | :--------------: |
|    把米从米袋倒进锅里     |    **你**     | 你写 C 代码（main 函数） |
| 但是——锅没洗、插头没插、电没通 |   **没人管**    |  单片机刚上电，硬件一片混沌   |
|  **洗锅、插电、按开关**   | **一个看不见的帮手** |     **启动文件**     |

**启动文件就是那个在你动手做饭之前，先把厨房收拾好、把电通好的"隐形帮手"。**

没有这个帮手，你写的 main 函数就像一个厨师走进没洗锅没插电的厨房——什么都干不了。

### 再打个比方：演戏

你想演一出戏（你的 C 程序），但上台之前：

- **舞台要搭好** → 堆栈指针 SP 设置好（给局部变量腾地方）
- **灯光音响要对** → 系统时钟配置好（72 MHz）
- **演员表要排好** → 中断向量表建好（各种中断来了找谁处理）
- **幕布拉开你才能出场** → 最后才跳转到你的 main()

**启动文件就是那个在幕后做完所有准备工作、然后喊"action"的导演。**

### 一句话大白话版

> **启动文件就是一开机先跑的一段"打杂代码"，它把芯片的"水电煤气"通好、把"家具摆好"，然后才叫你写的 main 函数出来干活。**

## 二、专业解释

### 2.1 启动文件是什么

启动文件（Startup File）是一个以 `.s` 结尾的**汇编语言源文件**，通常是芯片上电复位后 CPU 执行的**第一段代码**。它位于程序的**最底层**，是连接硬件和 C 运行时环境的桥梁。

位置关系：

```
芯片上电
    ↓
启动文件 (.s)  ← 最先执行，汇编写的
    ↓
SystemInit()    ← C 函数，配置系统时钟
    ↓
__main          ← C 库函数，初始化全局变量
    ↓
main()          ← 你写的用户程序
```

如果没有启动文件，CPU 上电后不知道栈在哪、不知道中断去哪里找处理函数、不知道要不要配时钟——直接跑 C 代码就会崩溃。

---

### 2.2 启动文件的六大核心作用

#### ① 定义中断向量表

中断向量表是一个 **32 位函数地址数组**，存放在 Flash 起始地址。每个中断/异常按编号对应一个表项，CPU 硬件自动查表跳转。

```asm
__Vectors       DCD     __initial_sp            ; 0x0000_0000  栈顶地址（不是函数）
                DCD     Reset_Handler           ; 0x0000_0004  复位中断 → 程序入口
                DCD     NMI_Handler             ; 0x0000_0008  不可屏蔽中断
                DCD     HardFault_Handler       ; 0x0000_000C  硬件错误 ⭐ 最常见
                DCD     MemManage_Handler       ; 0x0000_0010  内存管理错误
                DCD     BusFault_Handler        ; 0x0000_0014  总线错误
                DCD     UsageFault_Handler      ; 0x0000_0018  用法错误（除零等）
                ; ... 外设中断向量（USART1_IRQHandler、TIM2_IRQHandler 等）
```

向量地址计算公式：`0x00000000 + 异常号 × 4`

`DCD`（Define Constant Data）是 ARM 汇编伪指令——"在当前地址放一个 32 位值"。

#### ② 设置堆栈指针 SP

C 语言依赖栈来存放局部变量、函数参数、返回地址。启动文件在第一条指令执行前，将 SP 指向 RAM 的最高地址。

```asm
Stack_Size      EQU     0x00000400              ; 栈大小：1 KB
Heap_Size       EQU     0x00000200              ; 堆大小：512 字节

                AREA    STACK, NOINIT, READWRITE, ALIGN=3
Stack_Mem       SPACE   Stack_Size
__initial_sp

                AREA    HEAP, NOINIT, READWRITE, ALIGN=3
__heap_base
__heap_limit
```

> 默认 1 KB 栈对于简单工程够用，如果用了大量局部变量或深度函数嵌套，出现 HardFault 时可以考虑增大。

#### ③ 定义程序入口：Reset_Handler

Cortex-M3 内核规定：上电后从 `0x00000004` 取出地址作为 PC 的初始值——这个地址就是 `Reset_Handler`。

```asm
Reset_Handler   PROC
                EXPORT  Reset_Handler           [WEAK]
                IMPORT  __main
                IMPORT  SystemInit

                LDR     R0, =SystemInit         ; 加载 SystemInit 地址
                BLX     R0                      ; 调用 → 配置时钟
                LDR     R0, =__main             ; 加载 __main 地址
                BX      R0                      ; 跳转到 C 库初始化
                ENDP
```

#### ④ 初始化系统时钟

`Reset_Handler` 调用 `SystemInit()`（实现在 `system_stm32f10x.c`），将系统时钟从默认的 **8 MHz HSI**（内部高速振荡器）切换到 **72 MHz HSE**（外部晶振 + PLL 倍频）。

这就是为什么你写 main 时不需要手动配置时钟——启动文件已经帮你做了。

#### ⑤ 跳转到 C 运行时环境

`SystemInit()` 返回后，跳转到 C 库的 `__main`（注意不是你的 main），它负责：

1. **拷贝数据段**：将 `.data` 段的初始值从 Flash 复制到 RAM
2. **清零 BSS 段**：将所有未初始化的全局/静态变量置零
3. **初始化堆**：为 `malloc` 等动态内存分配做准备
4. **调用 `main()`**：终于执行你写的代码

#### ⑥ 提供弱定义的中断服务函数（WEAK 机制）

启动文件为每个中断提供了一个**弱定义的空函数**——函数体只有一条 `B .`（跳转到自身，即无限循环）：

```asm
NMI_Handler     PROC
                EXPORT  NMI_Handler             [WEAK]
                B       .                       ; 无限循环
                ENDP
```

- 如果你**没有**实现某个中断的处理函数 → 触发该中断时程序进入死循环（相当于死机）
- 如果你**自己写了**同名的函数 → 链接器自动用你的函数覆盖这个弱定义

> 这也是为什么你在 `stm32f10x_it.c` 里写中断函数时，**函数名必须和启动文件里的一模一样**——名字对不上就不会覆盖，中断来了照样死循环。

### 2.4 完整执行流程

```
芯片上电 / 复位
    │
    ▼
① 硬件从 0x0000_0000 读取 4 字节 → 装入 SP（栈顶指针）
    │
    ▼
② 硬件从 0x0000_0004 读取 4 字节 → 装入 PC（程序计数器）
    │   这个地址就是 Reset_Handler 的入口
    ▼
③ 跳转到 Reset_Handler 执行
    │
    ▼
④ 调用 SystemInit()
    │   配置 HSE → PLL → 72 MHz 系统时钟
    ▼
⑤ 跳转到 C 库 __main
    │   ① 拷贝 .data 段（Flash → RAM）
    │   ② 清零 .bss 段
    │   ③ 初始化堆
    ▼
⑥ 调用你写的 main()
    │
    ▼
⑦ 进入 while(1) 无限循环
```

### 2.5 常见问题

**Q：启动文件是汇编写的，我需要完全看懂吗？**

不需要。你只需要：
- 知道它的作用
- 会选正确的那一个（根据 Flash 容量）
- 会调整堆栈大小（Stack_Size / Heap_Size）

**Q：什么时候需要修改启动文件？**

- 栈溢出出现 HardFault → 增大 `Stack_Size`
- 做 IAP Bootloader → 修改向量表位置
- 需要在 main 前执行特殊硬件初始化 → 在 Reset_Handler 中添加

**Q：程序下载后不运行，90% 是？**

1. 选错了启动文件（如 C8 用了 hd.s）
2. 工程中根本没加启动文件
3. `SystemInit` 中时钟配置错误（晶振频率不匹配）
