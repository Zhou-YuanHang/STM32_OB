# 寄存器、寄存器映射、存储器和存储器映射
## 1. 寄存器（Register）

**寄存器**是 CPU 或外设内部的一种 **超高速存储单元**，通常宽度为 8/16/32/64 位。

- **CPU 寄存器**：`R0`~`R15`、`PC`（程序计数器）、`SP`（栈指针）—— 在 CPU 核内，用于暂存指令、地址、中间结果。
- **外设寄存器**：例如 `GPIO_ODR`（输出数据寄存器）、`USART_DR`（数据寄存器）—— 控制外设的行为。

> 关键：寄存器不在内存里，而是分布在 CPU 核或外设内部，CPU 通过 **地址** 访问它们。

```c
// 示例：写一个外设寄存器（伪代码）
*(volatile uint32_t*)(0x40020014) = 0x0000FF00;  // 直接往地址写值
```
---

## 2. 存储器（Memory）

**存储器**是能存储数据的硬件，包括：

| 类型 | 特点 | 例子 |
|------|------|------|
| **RAM** | 可读可写，掉电丢失 | SRAM、SDRAM、DDR |
| **ROM/Flash** | 只读（或可擦写），掉电保留 | 程序 Flash、EEPROM |
| **Cache** | CPU 内部的透明缓存 | L1/L2/L3 |

在嵌入式系统中，**存储器地址空间** 是一片连续的地址范围（如 `0x0000_0000 ~ 0xFFFF_FFFF`），不同区域映射到不同硬件。

---

## 3. 存储器映射（Memory Mapping）

**存储器映射** 是指把 **外设的寄存器** 或 **外部存储器** 映射到 CPU 的地址空间里，使 CPU 可以用 **访问内存一样的指令**（`LDR`/`STR`/`MOV`）来读写它们。

以 STM32F407 为例，地址空间布局：

```
0x0000_0000 - Flash (1MB)
0x2000_0000 - SRAM (192KB)
0x4000_0000 - 外设总线 (GPIO, USART, SPI, TIM…)
0xE000_0000 - Cortex-M 系统控制 (NVIC, SysTick…)
```

CPU 看到的是 **一个平坦的地址空间**，不管背后是 Flash、RAM 还是外设寄存器，CPU 只管发地址。

---

## 4. 寄存器映射（Register Mapping / Register Map）

**寄存器映射** 有两层含义：

### (a) 硬件层面

给每个寄存器分配一个 **偏移地址（offset）**，加上外设基地址就得到绝对地址。

例如 STM32 GPIOA 的寄存器映射：

| 寄存器 | 偏移 | 作用 |
|--------|------|------|
| `GPIOA_MODER` | `0x00` | 模式选择（输入/输出/复用/模拟） |
| `GPIOA_ODR` | `0x14` | 输出数据 |
| `GPIOA_IDR` | `0x10` | 输入数据 |
| `GPIOA_BSRR` | `0x18` | 位设置/复位 |

基地址 `GPIOA_BASE = 0x4002_0000`，所以 `GPIOA_ODR = 0x4002_0014`。

### (b) 软件层面（C 语言实现）

用 **结构体指针** 或 **宏定义** 把地址映射成变量名，让代码可读。

```c
// 方式一：宏定义
#define GPIOA_ODR  (*(volatile uint32_t*)(0x40020014))

// 方式二：结构体映射（最常用）
typedef struct {
    volatile uint32_t MODER;   // 0x00
    volatile uint32_t OTYPER;  // 0x04
    volatile uint32_t OSPEEDR; // 0x08
    volatile uint32_t PUPDR;   // 0x0C
    volatile uint32_t IDR;     // 0x10
    volatile uint32_t ODR;     // 0x14
    volatile uint32_t BSRR;    // 0x18
    volatile uint32_t LCKR;    // 0x1C
    volatile uint32_t AFR[2];  // 0x20
} GPIO_TypeDef;

#define GPIOA ((GPIO_TypeDef*)0x40020000)

// 使用
GPIOA->ODR = 0x0000FF00;
```

这种做法就叫 **寄存器映射** —— 把硬件手册里的寄存器布局翻译成 C 语言结构体。

好比说 GPIOA_ODR 的地址是 `0x40020014 `，但是要是直接这么写肯定不行，因为这不会被编译器看作为一个地址，所以使用 `(volatile uint 32_t*)` 强行将其转换为一个地址，前面的 `*` 就是地址的值，在通过 `#define GPIOA_ODR` 就完成了使用 `GPIOA_ODR` 直接读取到地址的值。这是方式 1 的逻辑，方式 2 逻辑差不多，就是通过了结构体指针来实现了这个效果

---

## 5. 实际工作流程（以点灯为例）

```
STM32F407 数据手册
    ↓
查阅 GPIOA 基地址 = 0x4002_0000，ODR 偏移 = 0x14
    ↓
C 语言寄存器映射
    typedef struct { ... } GPIO_TypeDef;
    #define GPIOA ((GPIO_TypeDef*)0x40020000)
    ↓
写寄存器
    GPIOA->MODER = 0x50000000;   // PA14 设为输出
    GPIOA->ODR   |= (1 << 14);   // PA14 输出高电平
    ↓
CPU 执行 STR 指令
    地址 0x40020014 → AHB 总线 → GPIO 外设 → 引脚电平变化
```

---

## 6. 一句话总结

| 概念 | 一句话 |
|------|--------|
| **寄存器** | CPU 或外设内部的高速存储单元，有固定功能 |
| **存储器** | RAM/Flash 等数据存储硬件 |
| **存储器映射** | CPU 给每个硬件（RAM/Flash/外设）分配一个地址范围，统一用访存指令访问 |
| **寄存器映射** | 把外设的寄存器布局翻译成 C 结构体/宏，让代码可读 |

### 层级关系

```
CPU 地址总线
  │
  ├─ 0x0000_0000 ~ 0x1FFF_FFFF  →  Flash（存储器）
  ├─ 0x2000_0000 ~ 0x3FFF_FFFF  →  SRAM（存储器）
  │
  └─ 0x4000_0000 ~ 0x5FFF_FFFF  →  外设总线
         └─ 0x4002_0000（GPIOA 基地址）→ 寄存器映射
                ├─ 0x00 MODER
                ├─ 0x04 OTYPER
                ├─ ...
                └─ 0x14 ODR  ← 这就是一个寄存器
```
